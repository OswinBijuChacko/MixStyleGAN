# Pin huggingface_hub to the 0.x API line. Both Gradio 4.x and 5.x
# (as of Q1 2026) still import `HfFolder` from huggingface_hub in
# their oauth.py — that name was removed in hub 1.0. Without this
# pin, every Space build crashes at `import gradio` with:
#   ImportError: cannot import name 'HfFolder' from 'huggingface_hub'
# 0.30 is HF Spaces' minimum (they install hf-transfer alongside).
huggingface_hub>=0.30,<1.0

# Python 3.13 removed `audioop` from stdlib; pydub (a Gradio dep)
# still imports it. audioop-lts is the standalone backport. Only
# pulled on 3.13+; ignored on the 3.11 we pin in the HF Space YAML.
audioop-lts; python_version >= "3.13"

# Pin numpy below 2.0 — diffusers/transformers wheels were compiled
# against numpy 1.x and break with the 2.x ABI ("dtype size changed,
# may indicate binary incompatibility"). Remove the upper bound once
# upstream wheels ship for numpy 2.x.
numpy>=1.24.0,<2.0

# Pin pydantic below 2.10. Pydantic 2.10+ emits `additionalProperties: true`
# (a bool) for dict types in JSON Schema. Gradio's gradio_client/utils.py
# parser assumes additionalProperties is always a dict, and crashes the
# /api_info endpoint with `TypeError: argument of type 'bool' is not iterable`
# when introspecting our generate() function (mask_editor is a dict).
# Removing this once gradio_client handles bool schemas (5.10+? unclear).
pydantic>=2.0,<2.10
